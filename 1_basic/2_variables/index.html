Variables

Global and local variables In vanilla JS (ES5):

Local variable (var)

  • Created by using the var declaration (var x).
  • Local variable can be initiated with the decleration (var x = "yellow").
  • Var decleration are processed before any code is executed (hoisting).
  • A redeclared variable dosen't lose its value.
  • A variable that has been declared but has not been initiated has the value of undefined.
  • Its scope is the current execution context (global or function).

Global variable

  • An undeclared variable is a global variable (x = 5).
  • Vanilla JS strict mode will throw an error for an undeclared variable.
  • Variable of the global scope become a property of the global object, Including the global variable that was declared with a var (i.e., window).

Constant and local variables in JS (ES6):

Local variable (let)

  • Created by using the let declaration (let x).
  • Local variable can be initiated with the decleration (let x = "yellow").
  • Can't be redclared.
  • A variable declared with let is not hoisted.
  • A variable that has been declared but has not been initiated has the value of undefined.
  • Its scope is the current enclosure context (global, function, loop, if, etc.).
  • Doesn't become a property of the global object.

Constant (const)

  • Created by using the const declaration (const x).
  • Must be initialized (const x = 8).
  • Its scope is the current enclosure context (global, function, loop, if, etc.).
  • A constant can't be reassigned.

JS variables naming rules and convention

  • A variable name can contain only letters, numbers, $ and '_'.
  • A variable name can't start with a number'.
  • Should only start with a lowercase letter.
  • Two words should be written in camelCase format.
  • The '_' shouldn't be the first or last character of the variable name.
